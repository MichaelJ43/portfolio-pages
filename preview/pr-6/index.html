How these pages are wired

These portfolio pages are static HTML at the edge, but they share the same small platform pieces I use across public projects. The skill chips on each card call out where those patterns show up in the work.

Static assets
Shared tokens, shell styles, primitives, analytics, and auth-header scripts load from static.michaelj43.dev.
Shared API platform
Client-side integrations point at api.michaelj43.dev so pages can stay static while calling common services.
Auth process
The header script talks to auth.michaelj43.dev and renders sign-in state without each page owning auth UI.
Analytics
A lightweight shared script records page usage with an app id, keeping instrumentation consistent across projects.
Domain routing
Custom domains route live apps and portfolio pages to the right static build, preview path, or service origin.
